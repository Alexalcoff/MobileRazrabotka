# MobileRazrabotka
Проект для предмета Мобильная разработка.

За время было сделано приложение для взаимодействия пользователя с алгоритмом, в приложение с помощью центральной кнопки загружается аудиофайл, после чего он отправляется на сервер, там производятся манипуляци, после чего в качестве json ффайла отправляет обратно на клиентское приложение. Там оно преобразуется в несколько лучших вариантов поожести треков. В самом приложении так же представлены разделы История и Настройки под сответствующими иконками, в настройках пока что просто заглушка. В разделе история представлены предыдущие запросы, их можжно переклчать через dropdown и листать в окне снизу. среди признаков передаваемых есть: название и степень похожести, они представлены в порядке убывания, в данный момент треков на один запрос - 15. Время запроса так же долгое, так как функция поиска на последнем этапе мало оптимизирована.

Запросы приложение передает на сервер, который использует программу расположенную по ссылке: https://github.com/Alexalcoff/audio-analysis-mobile.git   . Там просто анализ трека с учетом базы данных лежащей там же, по сути трек прогоняется через алгоритм, разбивается на первичные признаки, по типу грубой chroma, энергию, грубой mfcc, и их производных вычисленных численно, ну скорее протсо разность соседних, чтобы исследовать соотношение в передаваемых образцах. Дальше они сраниваются с помощью косинусного преобразования, ну чтобы скалярное произведение поделенное на две нормы было близко к единице. Ну и потом с помощью DTW (Dynamic Time Warping) анализируются выбранные первой выборкой 15 треков. На последнюю проверку и уходит много времени, даже если она сейчас немного оптимизирована. 

Проблема проекта сейчас заключается в том, что первичная проекта обычно всегда выбирает определенное подмножество из множества треков, так что допустим при выводе результатов всегда встречаются примерно 7 вариантов, из-за того, что они свойствами в большинстве своем похожи. Я пытался нормализовать свойства, чтобы они не сильно выбивались по нормам, но все равно находилось такое подмножество.


Так же считаю важным уточнить, что чтобы приложение работало нужно запустить FastAPI сервер, чтобы он принимал запросы. Запускаю его через ssh соединение, перехожу в соответствующую директорию, для дальнейшего запуска uvicorn, и после этого он может принимать запросы с выводом в приложение. 


На этом вроде все. Оформление не слишком официальное, но вроде полное.

А, ну и в дальнейшем планирую для себя добавить поддержку ссылков на треки, чтобы они выводили какое-то изображение, мб сделаю проигрывание части трека со спотифая, ну на те которые там есть, иногда база не пересекается со всем многообразием треков со спотифая. Наверное улучшу дизайн, добавл. настройки например звука, волн которые кнопка источает, сейчас не посчитал это важным.
